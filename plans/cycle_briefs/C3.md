# Cycle 3 brief — Text stack, color, and the divergence harness; R1 ships

**Layers:** L4 · **Envelope:** **R1 — Explainable Display Core ships** · **Kernel-touch expectation:** `none`

This is a brief. The C3 delta plan is authored after the C2 close gate passes, against the repository as C2 actually leaves it (Charter §6 cl. 1).

---

## 1. Scope boundary

**In.** Font parsing across the families PDF actually contains: TrueType/OpenType, CFF/Type 2, Type 1, CID-keyed and Type 0, and Type 3 procedure fonts (§20.3, §20.7–20.10). CMaps and ToUnicode, including variable-length codespace evaluation (§20.5–20.6). Budgeted glyph execution behind the `FontExecutionService` protocol that C2 shipped as an interface and C3 now implements (§30.5–30.6) — the format crates never depend back on the page VM. Widths, positioning, text rendering modes, and text clipping (§20.11–20.11.1). Text extraction as Workflow C, with the glyph-instance graph, Unicode evidence, and reading-order hypotheses kept as *hypotheses* (§5.3, §20.13–20.15). Base-font substitution under a resolver profile (§20.12). The ICC color pipeline proper, replacing C2's device-space-only posture (§19.5). The external-processor divergence harness (§24.4–24.9). The Observatory seed — **records and minimized witnesses only** (§28.11; Bet 21).

**Out.** The browser playground. Bet 21's Observatory needs the WASM surface, which lands at C4; C3 produces the data the playground will later display, and nothing that displays it. TrueType hinting's release envelope stays governed by its §35 default (unhinted/high-resolution first if necessary). Font subsetting and embedding are writer-side and belong to C4. Authoring-side shaping and layout (UAX #14/#29/#9) are C6 Track B — C3 owns *reading* text, not *setting* it. Separation/DeviceN and rendering intents remain C6.

**Why this boundary.** Text is where honest PDF engines go to die, and it is where MonkeyBee's central claim — that a result carries its own reasons for trust — is most testable. A glyph either maps to Unicode with evidence, or it does not and says so (§20.13). Landing the text stack *after* a working rasterizer means every text bug is a text bug, not a text-or-raster bug. And R1 ships here rather than at C2 because a display core that cannot show text is not a display core — it is a rasterizer with a promise.

## 3. The divergence harness is the campaign's first contact with other engines. Read this twice.

C3 runs external PDF processors as **experimental subjects** under the black-box oracle firewall (§4.6). Three laws govern it, and each has a different failure mode:

1. **Clean-room (§4.1, AGENTS rule 8).** Never clone, fetch, read, or cite their source or documentation. Not for reference. Not the README. The adapter layer collects success/failure, diagnostics, rendered pixels, extracted text, object inventories, and output hashes — and exposes no external header, API, or internal data structure to any production crate. Contamination cannot be undone, and it would void the artifact claim retroactively.
2. **No premature measurement (AGENTS rule 9).** Running an external engine to *classify a disagreement* is oracle use. Running it to *produce a comparative result* is measurement, and measurement before the day-zero discovery protocols are committed through the substrate permanently poisons the Q2 evidence chain. The C3 harness is a divergence instrument. It is not a scoreboard, and its outputs may not be framed as one.
3. **No majority vote (§24.5).** Agreement among external processors is evidence, not truth. Disagreements go to standards, analytic oracles, and adjudication — never to a show of hands. A reference image is a named processor's behavior, not the specification.

**Hard prerequisite.** C3's harness cannot run before day-zero item 3 (discovery protocols committed) exists. If the substrate has not materialized by C3, the harness either waits or runs in a mode that produces no comparative record at all — and that constraint is a Charter-level fact, not an implementer's judgment call.

## 6. Known risks

1. **The wild-tail gate is the cycle's real deadline.** It is the first time the artifact meets documents nobody curated. If the steward did not seal the corpus before implementation began (day-zero item 6), there is no held-out claim to make — after code exists, nothing new can become held-out (§28.3). This risk was created or eliminated on day 0 and cannot be repaired at C3.
2. **Glyph execution is an execution surface.** Charstring interpreters, hinting VMs, and Type 3 procedures execute attacker-supplied programs. They inherit §11's budget, cancellation, and panic law, and they belong in the fuzz inventory from the first commit (§27/G5).
3. **Font pathologies are a bottomless well.** The C3 plan must state which pathologies R1 refuses, page-locally and by name, rather than discovering the boundary in production.
4. **Divergence harness scope creep into measurement.** See §3. The pressure to "just check how we compare" will be real, and yielding to it is irreversible.
5. **ICC correctness has weak internal oracles.** Color needs analytic patches and independent profile corpora (§27/G2), or it becomes a subsystem nobody can falsify.
